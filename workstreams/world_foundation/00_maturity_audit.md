# WF-00.6 Maturity Audit Report

This artifact records the current operation-to-service-to-kernel-to-model
dependencies for all 15 registered tasks. It is generated by
`scripts/audit_maturity_report.py` and validated by
`src/chemworld/runtime/maturity_audit.py`.

The report distinguishes declared task maturity from the models actually
reachable through each task's `TaskRuntimeProfile`. WF-00.3 aligns the
equilibrium-characterization declaration with its reachable reaction, reactor,
pH, and UV/Vis providers; the fixed-T,P Gibbs solver remains an explicit
reference-only provider and is not declared as a runtime task dependency.
`minimum_actual_maturity` is the lowest provider maturity among the models
reachable from the task. A task with no physical provider route is reported as
`not_applicable`.

The WF-00.4 role audit also records `actual_provider_roles`,
`declared_provider_roles`, and `actual_role_partition` for `runtime`,
`runtime_fallback`, `diagnostic`, and `reference` providers. Reference-only
providers must never appear in an actual runtime partition; a violation fails
the role boundary audit. A diagnostic provider may be reachable for an
explicit diagnostic route but does not raise task maturity by itself.

WF-00.1 derives default task maturity modules from physical model operation
routes. Ledger-only operations no longer inherit reaction, reactor, or
instrument declarations merely because they occur in the same task. Remaining
declaration gaps in the JSON are therefore actionable differences between
task-specific contracts and actual reachability, rather than unconditional
shared-module noise.

Reference boundary:

- ChemWorld `TaskSpec`, `TaskRuntimeProfile`, `DomainServiceRegistry`,
  `OperationKernelRegistry`, and `ModelReachabilityRegistry` are the audited
  contracts.
- External `reference_repos/` are not used as runtime implementations by this
  report. Their fixed commits remain read-only comparison boundaries for the
  underlying physical modules.

Regenerate and validate:

```powershell
$env:PYTHONPATH = "src"
python scripts/audit_maturity_report.py
python scripts/manage_claims.py check
```

The committed JSON is a reproducible snapshot. Its task contract hashes,
runtime profile hashes, dependency graph, maturity values, and evidence paths
must be revalidated whenever a task or route contract changes.

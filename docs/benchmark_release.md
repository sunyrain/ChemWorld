# 当前科学状态

如果你只想知道 ChemWorld “现在能不能用”，答案分成两部分：

- **可以用来开发和研究**：环境、任务合同、资源账本、轨迹回放和评价控制已经可运行。
- **还不能当作定型排行榜**：完整方法矩阵、隐藏世界泛化和独立复现仍未共同完成。

!!! info "一句话状态"
    当前是 **controls ready、formal evidence incomplete** 的 benchmark candidate。可以开展 Agent
    开发、教学和诊断实验；正式 leaderboard、SOTA、真实化学发现及 RL/LLM 优越性结论仍需等待
    完整证据。

## 经典优化：分数变高，不代表方法整体更好

无约束 structured GP 在分配、结晶、蒸馏和连续流四项任务上都提高了声明的主指标，但在连续流、
结晶和蒸馏上显著增加操作风险预算超限率。完整目标—安全—成本规则因此失败。这一结果说明只看
产率或总分会把危险的优化策略误判为更好。

修复后的 Safe-GP 学习单次实验的峰值操作风险，并使用类别材料表示和风险上置信界。冻结后完成
四任务、三方法、20 个配对世界、每次 40 个完整实验的确认切片；240 条轨迹全部通过第二次回放。

| 任务 | Safe-GP − random 主指标效应 | 区间 | SESOI | safety/cost | 联合任务规则 |
| --- | ---: | ---: | ---: | --- | --- |
| 分配发现 | +0.036579 | [0.026105, 0.047188] | 0.0292 | 通过 / 通过 | 通过 |
| 反应—结晶 | +0.102475 | [0.082343, 0.122691] | 0.038827 | 通过 / 通过 | 通过 |
| 反应—蒸馏 | +0.049918 | [0.034127, 0.066079] | 0.0200 | 通过 / 通过 | 通过 |
| 连续流优化 | +0.018752 | [0.013144, 0.023698] | 0.0200 | 通过 / 通过 | **失败** |

四项目标方向均为正，安全和成本规则全部通过；但连续流效应没有达到预注册最小实质效应。因此
不能声明 Safe-GP 在完整四任务规则下优于 random。

一个随后进行的五世界开发诊断比较了风险置信系数 2.0、1.5 和 1.0。系数 2.0 同时得到最高平均
连续流转换率、最低风险超限率和最低成本；降低系数不是可靠修复。该小样本诊断只用于拒绝这一
简单调参方向，不改变前述确认结论。

## 强化学习：训练流程跑通了，排名还没有

SAC 在连续流 Train 环境完成了精确 100,000 步，并保留 20k、40k、60k、80k 和 100k checkpoint
与 replay buffer。最终策略在开发环境完成 20/20 episodes、无非法动作或领域失败；10/10 标准轨迹
可精确回放。

学习曲线并不单调：80k checkpoint 的开发均值为 0.023782，100k 为 0.010552，只保留前者的
44.37%。这证明“更多训练步一定更好”不成立，也说明正式 RL 结果必须用多 seed 的开发集合选择
checkpoint。当前结果只有一个任务、一个模型 seed，不能用于 RL 方法排名。

## 隐藏机理：环境变化可执行，Agent 迁移仍待验证

三类反应任务使用速率律与网络拓扑族；分配、电化学和平衡使用各自 provider 消费的构成律族。
在 5 个世界 × 5 个配方探针下，9/9 任务—模式组合满足：

- 干预确定且使用 opaque 公共标识；
- 得分变化达到可辨识下限；
- 90 分位变化不超过非灾难上限；
- 过程质量衡算误差在容差内；
- 缺少或篡改精确干预上下文时回放失败关闭。

这些是环境控制证据，不是机理发现或迁移结果。正式实验仍需冻结不重叠的 Train/Dev/Bench 机理族，
比较 Agent 是否识别变化并将经验迁移到未见 family。

## LLM：交互链已经具备，真实模型矩阵仍缺失

官方 adapter 支持 DeepSeek V4 Pro/Flash、逐操作多轮决策、跨实验记忆、token/费用/重试账本和失败
保留。assigned/masked 谱图消融只删除谱图、峰、通道与 assignment；质量衡算、端点、成本、预算和
约束保持一致。环境不会替模型修复动作、自动终止或补做终测。

当前没有真实 provider 轨迹。离线 fake client、stub 或 replay 只能证明协议和回放链工作，不能作为
LLM 性能证据。私有逐字思维链不请求也不保存；公开轨迹只保留 evidence、hypothesis、uncertainty、
spectrum interpretation 与简短 rationale。

## 不同任务现在承担什么角色

| 角色 | 任务 | 用途 |
| --- | --- | --- |
| 软件回归 | assay、purification、partition 等稳定切片 | API、回放和发布链检查 |
| 研究核心候选 | partition、crystallization、distillation、flow | 正式跨方法比较候选 |
| 探索任务 | electrochemical、equilibrium | 主指标有效性仍需确认 |
| 教学与能力切片 | 其余注册任务 | 课程、接口和诊断，不进入主排名 |

任务已注册、没有旧 fallback 或单元测试通过，都不等于 benchmark 有效。

## 发布前还差什么

1. 使用统一 vNext 风险策略重跑完整方法矩阵。
2. 完成多 seed PPO/SAC 训练、开发选择和冻结评测。
3. 完成 Pro/Flash × assigned/masked 的真实 LLM 配对运行。
4. 运行 Agent 机理识别、跨 family 迁移与隐藏世界泛化实验。
5. 完成独立 reference portfolio，避免把被评方法的最好值称为 oracle。
6. 完成私有评测、正式 exploit 结果与干净安装独立复现。
7. 冻结可引用版本、归档与论文元数据。

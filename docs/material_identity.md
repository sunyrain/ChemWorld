# 材料名称代表什么

界面里会出现 Water、Ethanol、Toluene 和 Catalyst A–D。这里需要分清两件事：名称可以对应真实
材料身份，但它在任务中的动力学或分配效应仍可能来自虚拟世界参数。ChemWorld 因此同时保留稳定
机器编码、面向人的名称和独立模型证据。

## 当前可以选择的材料

| 动作值 | 使用者看到的名称 | 身份状态 | 当前反应任务中的作用 |
| --- | --- | --- | --- |
| solvent `0` | Water · H2O | 参考库身份与性质包 | 校准的类别效应 |
| solvent `1` | Ethanol · C2H6O | 参考库身份与性质包 | 校准的类别效应 |
| solvent `2` | Acetonitrile | 真实名称；尚未进入精选性质子集 | 校准的类别效应 |
| solvent `3` | Toluene · C7H8 | 参考库身份与性质包 | 校准的类别效应 |
| extractant `0..3` | 与 solvent `0..3` 相同的材料目录 | 相同身份边界 | 驱动分配系数与萃取风险 |
| catalyst `0..3` | Catalyst A..D | 匿名 benchmark 配方 | 潜在类别活性曲线 |
| reagent | Anonymous limiting reagent | 机理角色，不是物料身份 | 任务机理中的限制反应物 |

这里的关键区别是：`Ethanol` 确实标识乙醇，但当前反应任务中“溶剂 1 对速率的影响”仍来自
任务校准的类别参数，并不是从乙醇物性自动推导出来的。Catalyst A 也不能解释成 Pd/C、
Ni 或任何现实催化剂。

萃取剂复用同一稳定材料目录，但拥有独立于反应溶剂的设备状态。其代码会进入液液分配与风险模型，
不是仅供显示的标签；同一实验内追加萃取剂时类别保持不变，体积累计。这里的分配效应仍是经过校准的
benchmark 类别规律，不等同于从完整混合物物性预测出的真实萃取数据。

## 在 Action 中怎样填写

JSON 轨迹和 Gym 向量使用稳定数字值，避免改名导致历史结果失效：

```json
{"operation": "add_solvent", "solvent": 1, "volume_L": 0.04}
```

事件接口也接受语义名称，并在进入环境前规范化为相同的数字值：

```json
{"operation": "add_solvent", "solvent": "ethanol", "volume_L": 0.04}
```

新轨迹中的萃取剂也应使用明确代码或名称。早期轨迹使用的 `"extractant": "organic"` 仅作为兼容
别名保留，并固定规范化为 Toluene（代码 `3`）；它不表示一个额外的、身份未定的“有机相”选项。

Student Lab 的选择器、Agent action schema 和实验历史会同时显示名称；未知名称和越界索引会被
明确拒绝，不再静默截断。这样既保留协议稳定性，也避免学生和模型凭空记忆 `0/1/2/3`。

## 哪些性质来自参考库

精选参考层目前覆盖 water、ethanol、acetone、toluene、methane 和 carbon dioxide。每个组分由
不可变的 `ComponentIdentityRegistry` 管理，并记录规范 ID、别名、CAS、InChI/InChIKey、分子式、
来源标识和内容摘要。性质包包含经过本地参考测试的 DIPPR101 蒸气压相关式和 Poling 理想气体
热容相关式。

这套参考层的用途是提供可审计身份和限定范围内的物性计算。它不会自动把匿名反应网络升级为
真实反应，也不会为尚未验证的温度、相态或混合物条件背书。

## 什么时候才能使用真实催化剂或反应名称

为一个具名反应发布真实材料标签之前，至少需要同时提供：

1. 反应物、产物、溶剂和催化剂的稳定身份及来源；
2. 有适用范围和来源的动力学、热力学及相平衡参数；
3. 与这些材料一致的仪器响应或真实数据映射；
4. 独立回归、守恒、边界条件和外部参考验证；
5. 明确版本化的任务卡、机理摘要与已知限制。

在这些证据齐备前，保留 Catalyst A–D 和匿名反应物，比替换成看似专业的现实名称更科学。
